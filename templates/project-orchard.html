Interactive Orchard Visual Design

Interactive Visual Design • Eclipse
IAT265 – Final Individual Project (Simon Fraser University)

Overview

This project is an interactive visual design created in Eclipse. It explores how different elements respond to user input and generate visual changes through interaction. Using an orchard theme, the project guides the user through a sequence of actions, from digging and planting to growth and harvest.

Tools

Eclipse (Java)

Process

Concept and Planning

This project began by defining how multiple visual elements could interact within a single system. I identified key objects such as the shovel, seed, kettle, fertilizer, plant, and orchard background. Each element was assigned a specific role so the project could function as an interactive system rather than a static illustration.

Interaction Design

I developed a storyboard to structure the interaction flow. The sequence includes digging soil, planting a seed, watering it, waiting for growth, applying fertilizer, and finally harvesting fruit. This helped define how each user action would produce a clear visual response.

Implementation

In the coding stage, I translated the planned interactions into working behaviors in Eclipse. I first implemented the shovel interaction, allowing it to be selected, follow the mouse, rotate, and return to its original position. From there, I extended the system to include planting, watering, growth, and harvesting.

Iteration

Several iterations were required to refine interaction timing and clarity. Early versions did not clearly communicate the correct order of actions, so I adjusted visual feedback and interaction states to guide the user more effectively. These refinements improved both usability and overall flow.

Challenges

One of the main challenges was controlling the interaction sequence. Because actions needed to happen in a specific order, I had to carefully manage state changes and restrict when elements could respond.

Another challenge was translating storyboard ideas into interactive behavior. Actions that appeared simple in sketches required more detailed logic in code, especially for mouse interaction, object movement, and visual transitions.

Design Decisions

The project focuses on clear visual feedback. Each interaction produces a visible change, helping users understand what is happening and what to do next.

I also kept the system simple by limiting the number of elements and giving each one a specific role. This helps maintain clarity while still demonstrating layered interaction.

Reflection

This project helped me understand how interactive systems are built through connected actions, timing, and feedback. Planning interactions through storyboards made the coding process more structured and manageable.

If I continue improving this project, I would add smoother transitions, improve visual polish, and refine interaction feedback. This project reflects my interest in interactive design and my ability to combine planning, coding, and visual storytelling.
