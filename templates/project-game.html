2D Shooting Game

Game Development • Processing

Overview

This project is a 2D shooting game developed in Processing. The player can move in all directions and shoot enemies while avoiding damage. The game includes enemy waves, boss battles, pickups, and a scoring system. The goal is to survive and defeat all enemies and bosses.

Process

The project was developed directly in Processing, starting with basic player movement and shooting mechanics. At the beginning, the game only included simple enemies moving downward, but the gameplay felt repetitive.

To improve the experience, I gradually added more systems. Enemies were updated with collision detection and shooting behavior, and a scoring system was introduced. Boss enemies were later added to create a stronger sense of progression and challenge.

The structure of the code was an important part of the process. I designed a class system using inheritance, where a base class handled shared movement logic. This allowed me to extend it into Player, Enemy, BossEnemy, and Pickup classes, making the code more organized and reusable.

Throughout development, I continuously tested and refined the gameplay. Mechanics such as enemy speed, bullet patterns, and difficulty scaling were adjusted multiple times to improve balance and player experience.

Challenges

One of the main challenges was managing the complexity of the code as more features were added. Handling interactions between players, enemies, bullets, and pickups became increasingly difficult.

To solve this, I reorganized the system using inheritance and polymorphism. This allowed different objects to share behavior while maintaining their own functionality, making the code easier to manage and extend.

Another challenge was balancing the gameplay. Early versions were either too easy or too chaotic. I adjusted enemy spawn rates, bullet speed, and boss difficulty multiple times to create a more engaging experience.

Design Decisions

The game was designed to gradually increase difficulty, allowing players to learn mechanics step by step. Basic enemies introduce the gameplay, while bosses add complexity and challenge.

Pickups such as health and weapon upgrades were added to support the player and introduce strategy. Visual elements such as score and health indicators help players understand their current state.

Reflection

This project helped me understand how to structure a larger program and manage multiple systems. I learned how object-oriented design improves efficiency and flexibility in game development.

If I continue improving this project, I would refine enemy behavior and add more gameplay variation. This project reflects my interest in game development and demonstrates my ability to combine programming, system design, and gameplay mechanics.
